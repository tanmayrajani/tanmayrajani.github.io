What is JSON?


   ✎   Format for sharing data

   ✎   Derived from JavaScript

   ✎   Language independent

   ✎   An alternative to XML


Symbol of JSON

How to write JSON?

  •   A JSON file is made of 'keys and value pairs' or 'ordered lists' (arrays)
  •   JSON keys wrapped in "quotes"
  •   JSON keys can be any valid string
  •   Filename Extension: .json
  •   JSON file should be validated before use; or data won't load if any single mistake is left in the format.

    (There are enormous websites/addons/extensoins/softwares available in order to write efficient and validated JSON.)

How to write JSON?

✎   JSON values have to be
one of the six data types:
strings, numbers, objects,
         arrays, booleans or null

✎   Internet Media Type
(MIME Type): application/json
Example of JSON

Advantages:

  • Easy to read (Human readable format)
  • Simple Structure
  • JSON is a valid subset of JavaScript, Python, and YAML
  • Easy to parse, process and validate in multiple languages
  • JSON can be mapped more easily to object-oriented systems.

Disadvantages:

  • It's data-exchange-format and not a document-exchange-format. Hence, it can only handle six basic 'data' types and not images,charts,graphs and other similar 'document' formats.
  • Originated in 2002. Yahoo! and Google started using JSON in 2005-06. So, it will take some time to become widely adopted.

JSON  v/s  XML

  •   "XML is like violence. If it doesn't solve your problem, you're not using enough of it."
  •   JSON is a better data exchange format. XML is a better document exchange format.
  •   XML allows you to store any document type you can come up with.
  •   JSON is much easier for human to read than XML. It is easier to write, too.
  •   JSON is processed more easily because of its simpler structure.

Language Support

  •   Currently around 61 programming languages/platforms have got third party libraries/APIs or workarounds to use JSON data-exchange format.
  •   This list contains all popular programming languages including C, C++, C#, Java, PHP, Python, Perl, Ruby, VBScript, CoffeeScript, MATLAB, ActionScript, Go and of course JavaScript.

JSON in JavaScript

  • There are basically two possibilities:
    1) JSON file is lying on the same server as JS file.
     2) JSON file is lying on some other server.
  • For former, we've got two alternatives: 1) Use AJAX calls
     2) Use AJAX with jQuery
  • For latter, there is only one alternative: 1) JSONP
